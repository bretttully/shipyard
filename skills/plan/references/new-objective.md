# NEW objective procedure

1. Light scope scan: direct read for a small surface; otherwise use `sy:sweep` enough to ask sharp questions.
2. Interview backwards from destination: objective/why, done-at-the-top, hard constraints, priorities. One shape-changing question at a time.
3. Once nameable, suggest the user run `/rename plan: <objective-slug>`.
4. Draft the Epic body with:
   - North Star — objective, why, done-at-the-top;
   - Context / constraints;
   - Out of scope.
5. Show the draft; create only on explicit approval. Use the `tracker` skill (`/sy:tracker`); new Epic starts `backlog`. If a portfolio parent exists, use the tracker's re-parent operation.
6. Continue with `roadmap-shaping.md`.
